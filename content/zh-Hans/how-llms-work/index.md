# LLM 到底是怎么工作的

在用 LLM 构建应用之前，你需要先搞清楚 LLM 实际在做什么——从机制上、从一次 API 调用的层面去理解。前端开发者对 LLM 的大部分困惑（为什么它会忘事？为什么这么贵？为什么同一个问题答案每次都不一样？为什么有上下文长度限制？）都源于缺少这一层基础。

本章就是这层基础。六个核心概念：

1. [Token](./tokens)——模型实际读取的内容
2. [下一个 token 预测](./next-token-prediction)——LLM 的全部机制
3. [从续写到对话](./completion-to-conversation)——一个"只会续写"的模型如何聊天
4. [多轮对话](./multi-turn)——模型是无状态的，是客户端在重放历史
5. [上下文窗口](./context-window)——1M tokens 到底是什么概念
6. [采样](./sampling)——温度、top-p，以及为什么同样的 prompt 每次得到不同的答案

把这六个概念想清楚之后，本指南后续的内容（RAG、Agent、微调、KV cache、GPU 选型）就都变成了"这一层在基本循环之上的什么位置"的问题。本章刻意跳过了 Transformer 架构的内部细节——重要的是**模型在做什么**，而不是它内部的矩阵乘法是怎么排列的。

收尾：[串起来看](./putting-it-together)——一次完整的 API 调用从头到尾，再加上指向后续章节的索引。
