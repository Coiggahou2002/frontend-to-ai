# 4. DPO 及其变体

更简单的偏好学习。

## DPO 的核心直觉

2023 年，Rafailov 等人提出了一个关键洞察：**语言模型本身就是一个隐式的奖励模型**。你不需要单独训练一个奖励模型，然后用 RL 去优化——可以直接用偏好数据训练语言模型。

DPO 的数学推导从 RLHF 的最优策略出发，通过变量替换推导出一个闭式解。最终的训练目标变成了一个简单的分类损失：给定一对"好回答"和"坏回答"，让模型增加好回答的概率、降低坏回答的概率。

```
给定一对偏好数据 (x, y_w, y_l)  -- y_w 优于 y_l

L_DPO = -log sigmoid(
    beta * ( log pi_theta(y_w|x) - log pi_ref(y_w|x) )
  - beta * ( log pi_theta(y_l|x) - log pi_ref(y_l|x) )
)
```

不需要奖励模型。不需要 PPO 循环。只是一个针对对数概率差的二元交叉熵损失，对照一个冻结的参考模型。

## RLHF 与 DPO 的流程对比

| 维度 | RLHF (PPO) | DPO |
| --- | --- | --- |
| 训练流程 | SFT -> RM -> PPO | SFT -> DPO |
| 需要的模型 | 4 个（策略、奖励、参考、价值网络） | 2 个（策略、参考） |
| 数据类型 | 排序数据 + 在线采样 | 离线配对偏好数据 |
| 训练稳定性 | 不稳定，需要大量调参 | 相对稳定 |
| 计算成本 | 高 | 中等 |
| 在线探索 | 有（通过采样新回答） | 无（只用静态数据） |

DPO 的最大优势是简化：不需要训练奖励模型，不需要复杂的 RL 循环，用一个标准的训练循环就能完成偏好优化。

## DPO 的局限

DPO 并非 [RLHF](./rlhf) 的完美替代：

- **数据分布敏感**：DPO 使用离线数据，如果训练数据的分布与模型当前策略差距太大，效果会打折
- **过拟合风险**：在小数据集上容易过拟合，特别是当"坏回答"并不是真的很差时
- **缺乏在线探索**：PPO 能在训练过程中采样新回答并获得反馈，DPO 只能从静态数据中学习

## DPO 的变体家族

DPO 的简洁框架催生了一系列变体，每个解决特定的痛点：

| 方法 | 核心改进 | 适用场景 |
| --- | --- | --- |
| IPO | 用 MSE 损失替代 sigmoid，增加鲁棒性 | 噪声偏好数据，坏回答不是很差 |
| KTO | 支持非配对数据（只需"好/坏"标签） | 难以获取配对偏好数据的场景 |
| SimPO | 去掉参考模型，用长度归一化对数概率 | 显存有限，追求效率 |
| ORPO | 将 SFT 和偏好优化合并为一步 | 希望减少训练阶段 |

## 选择建议

一个实用的选择框架：有充足计算资源和在线采样能力时，PPO/GRPO 通常效果更好；资源有限、追求简单时，DPO 或 SimPO 是合理选择；只有非配对数据时，KTO 是唯一选项。
