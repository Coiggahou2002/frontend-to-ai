# 5. GRPO 与推理强化

让模型学会深度思考。

## 从 DeepSeek-R1 说起

2025 年初，DeepSeek 发布了 R1 模型，展示了一个令人震撼的发现：通过纯 RL 训练（不需要人工编写的推理示例），模型可以自发涌现出链式思考（Chain-of-Thought）能力。这背后的核心算法就是 GRPO。

## GRPO 的核心创新

GRPO（Group Relative Policy Optimization）最初在 DeepSeekMath 论文中提出，它对 [PPO](./rlhf) 做了一个关键简化：**去掉价值网络（critic），用组内相对比较来估计优势函数**。

具体来说，对于每个输入问题，GRPO 会让模型生成一组回答（通常 64 个），然后用奖励函数（或验证器）给每个回答打分。每个回答的优势值不是绝对的，而是相对于这一组的平均水平：

```
advantage_i = (reward_i - group_mean) / group_std
```

这个设计有两个好处：一是完全消除了价值网络，节省了大量显存；二是组内归一化提供了自然的基线，减少了梯度方差。

```mermaid
flowchart LR
    A[Prompt x] --> B[策略采样 G 个回答]
    B --> C[逐个打分: reward_i]
    C --> D[组内归一化: A_i = (r_i - mean) / std]
    D --> E[PPO 风格的策略更新]
    E --> A
```

## RLVR：用验证器替代人类

GRPO 最强大的应用场景是 **RLVR（Reinforcement Learning with Verifiable Rewards）**。在数学和代码任务中，你不需要人类来判断回答好不好——单元测试、证明检查器或数学验证器可以提供完全客观的奖励信号。

这使得训练可以完全在线进行：模型生成回答 -> 验证器打分 -> 更新策略 -> 生成新回答，循环往复。DeepSeek-R1 正是通过这种方式，在数学和代码推理任务上实现了快速提升。

关键一点：RLVR 整个绕开了奖励模型的标注流水线。奖励函数就是*代码*，而代码不会跟自己有分歧。

## DAPO：解决推理训练的不稳定性

在大规模推理训练中，GRPO 仍然面临稳定性问题。ByteDance 提出的 DAPO 通过四个技术来解决：

1. **防止熵坍缩**：避免模型输出变得过于确定，丧失探索能力
2. **动态批次过滤**：丢弃无信息的训练样本
3. **token 级梯度计算**：比序列级更精细的梯度信号
4. **长度奖励调整**：避免模型学会"写更长的回答来获得更高奖励"

## 推理强化的边界

RLVR 的效果在有验证器的任务上非常显著，但它有明确的适用边界：

- **适合**：数学推理、代码生成、形式化证明、结构化输出
- **不太适合**：开放式写作、创意任务、主观问答
- **正在探索**：多步规划、工具使用、代理行为

不论你是在做 SFT、RLHF、DPO 还是 GRPO，决定胜负的很少是算法本身，而是数据。这正是下一节的主题。
