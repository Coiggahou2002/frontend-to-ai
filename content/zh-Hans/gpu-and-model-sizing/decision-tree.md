# 5. 决策树

把前四节内容拧成一张图：从"要部署什么模型"一路走到"买哪张 GPU"，再用两步法收尾。

## GPU 选型决策树

```
要部署什么模型？
│
├── Embedding 模型（≤8B）
│   └── FP16 显存 ≤ 16 GB
│       └── 选 1x A10 (24GB)
│           └── 选最便宜的 CPU/内存配置即可
│
├── LLM ≤ 14B
│   └── FP8 显存 ≤ 14 GB
│       └── 选 1x A10 (24GB)
│
├── LLM 14B-32B
│   └── FP8 显存 16-38 GB
│       └── A10 装不下 → 选 1x A100 80GB
│
├── LLM 32B-70B
│   └── FP8 显存 32-70 GB
│       └── 选 1x A100 80GB（FP8/INT8）
│       └── 或 2x A100 80GB（FP16）
│
└── LLM > 70B（如 235B MoE）
    └── FP8 显存 > 100 GB
        └── 多卡 A100 或 H100 → 成本极高，先考虑是否真的需要
```

## 总结：两步选型法

1. **先算显存**：参数量 x 精度对应的字节数 x 1.2 = 所需显存
2. **再选卡**：找能装下这个显存的最便宜的单卡方案

整章压缩成两行就这样。本章其它的经验法则——"FP8 是性价比甜点"、"既然已经买了 A100，跑 32B 跟跑 14B 钱一样"、"MoE 买的是速度不是省显存"、"能单卡就别多卡"——都是为了把这两步执行好。
