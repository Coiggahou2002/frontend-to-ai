# 1. GPU 心智模型

## 为什么大模型需要 GPU

CPU 擅长复杂逻辑和分支判断（少量核心，每个核心很强），GPU 擅长大规模并行计算（数千个小核心同时运算）。大模型的本质是海量矩阵乘法——正好是 GPU 的强项。

类比：CPU 像一个顶级厨师（做什么菜都行，但一次只能炒一盘），GPU 像 5000 个学徒同时切菜（每个人只会简单操作，但并行起来速度极快）。

这也是为什么"用 CPU 跑 LLM"理论上可行、但只要模型超过几个 B 就基本没法用：CPU 必须一两个核心一组地把那些矩阵乘法做完。GPU 50ms 跑完的一次前向传播，CPU 可能要 10 秒。这是架构上的根本错配，不是优化能补的。

## 这意味着本章接下来要讲什么

继续往下读之前，需要先内化的一个事实：**模型运行时，参数必须常驻 GPU 内存**。不是磁盘里、不是系统 RAM 里、也不是"按需加载"——全部、全程都得在。

GPU 内存叫 **VRAM**（Video RAM，显存），是直接挂在 GPU 芯片上的一块独立、远比系统内存更快的内存。它也比系统 RAM 小得多、按 GB 算贵得多。一台典型的云服务器可能有 128 GB 系统 RAM，但 GPU 上只有 24 GB VRAM。这种不对称就是"GPU 选型"成为一个真问题的根本原因：你必须把模型塞进那块小而快的内存。

所以本章每一节都会反复回到同一个问题：

> 我要跑的这个模型，需要多少 GB VRAM，哪张 GPU 至少能装得下？

剩下的一切——多卡、MoE 的怪癖、vCPU 和磁盘配置、不同 GPU 档位之间的成本悬崖——都是这一个计算的下游。
