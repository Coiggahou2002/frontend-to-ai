# GPU 与大模型选型

读到[第 4 章（Agent 与编排）](../agents-and-orchestration)结尾，你已经能围绕模型 API 构建系统了。从这里开始，本书往 API 这条线下面走——进入硬件与服务层。本章是第一步：**要把一个模型跑起来，到底需要多少 GPU、花多少钱？**

这章的视角是反着来的，跟厂商文档相反。我们不从 GPU 型号开始，而是从你想跑的模型开始，先算它要多少显存，再挑能装下的最便宜的卡。一旦这个习惯养成了，剩下三章（基础设施栈、KV cache、推理并发）讲的不过是"模型已经装进了 VRAM"之后的优化。

五个小节：

1. [GPU 心智模型](./gpu-mental-model)——为什么 LLM 需要 GPU，以及为什么 VRAM 是初期唯一重要的指标
2. [VRAM 与精度](./vram-and-precision)——参数量到 VRAM 的换算公式、FP16/FP8/INT4 各自的取舍、以及云上能租到的 GPU 型号
3. [多卡、MoE 和其他](./multi-gpu-and-moe)——单卡装不下时怎么办、MoE 模型藏的陷阱、以及非 VRAM 维度（系统 RAM、vCPU、磁盘）也要算
4. [模型尺寸怎么选](./choosing-model-size)——2026 年开源模型概览、能力分档、A10 到 A100 的成本悬崖、最终怎么决定
5. [决策树](./decision-tree)——从"要部署什么模型"一路走到"买哪张 GPU"的流程图，并用两步选型法收尾

目标不是把数字背下来，而是养成一个反射：**模型尺寸 x 精度 x 1.2 = VRAM，然后挑卡**。这个反射建立起来，后面几章基础设施的内容就顺理成章了。
